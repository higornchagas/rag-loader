[faq_crossramp_definicao]
O que é o Crossramp da NoxPay?
O NoxPay Crossramp é uma solução de pagamentos que converte moeda fiduciária (BRL) em criptomoedas (como USDT/USDC) utilizando o PIX como meio de pagamento.
O fluxo é completo e integrado: identificação → cotação → pagamento PIX → liquidação → notificação via webhook.

[faq_checkout_white_label]
É possível personalizar a página de pagamento (White Label)?
Sim. Embora o checkout padrão ocorra em ambiente seguro da noxpay.io por razões de compliance e biometria, é possível exibir o logotipo da loja e o resumo do pedido para reforçar a identidade visual do lojista.

[faq_tempo_biometria]
Quanto tempo o usuário tem para concluir a biometria facial?
Caso o motor de risco solicite biometria, o usuário possui até 10 minutos para concluir o processo.
Esse prazo acompanha a validade do QR Code PIX.
Se o pagamento ocorrer mas a biometria não for concluída dentro do prazo, o valor é estornado automaticamente.

[faq_pix_bancos_compatibilidade]
O sistema é compatível com todos os bancos brasileiros?
Sim. Como a NoxPay opera via PIX (Sistema de Pagamentos Instantâneos do Banco Central), qualquer banco ou instituição que ofereça PIX é compatível.

[faq_withdrawal_api]
Existe API para automação de saques?
Sim. Para lojistas de maior volume, disponibilizamos endpoints específicos que permitem programar saques automaticamente para wallets previamente cadastradas.

[faq_checkout_modelos]
A NoxPay oferece checkout transparente e redirecionado?
Sim. Existem dois modelos:
1. Checkout Redirecionado (padrão): usuário é direcionado para ambiente seguro da noxpay.io.
2. Checkout Transparente: QR Code e Pix Copia e Cola podem ser integrados diretamente no site do lojista via API.

[faq_declaracao_contabil]
Como o lojista deve declarar a entrada desses ativos?
A NoxPay fornece relatórios detalhados de todas as conversões e emite Nota Fiscal referente às taxas de intermediação.
Recomenda-se que o lojista registre como "Recebimento de Ativos Digitais via Pagamento" e utilize os extratos para conciliação contábil.

[faq_saques_automaticos]
É possível configurar saques automáticos?
Atualmente não. A funcionalidade está no roadmap, mas no momento os saques são manuais ou via API específica.

[faq_refund_estorno]
Como funciona o estorno (refund)?
Como os ativos digitais são entregues ao lojista, o estorno deve ser gerido pelo próprio lojista.
A NoxPay não realiza automaticamente o caminho reverso (Cripto → BRL) para evitar exposição à volatilidade.

[faq_pix_agendado]
A NoxPay aceita PIX agendado?
Não. Para garantir cotação travada e liquidação imediata, apenas pagamentos instantâneos são aceitos.

[faq_med]
Como a NoxPay lida com o MED (Mecanismo Especial de Devolução)?
Possuímos equipe de Compliance dedicada.
Caso o banco solicite devolução por suspeita de fraude, analisamos logs e biometria.
Se a fraude for confirmada antes do saque, o valor pode ser retido para devolução.

[faq_biometria_todas_vendas]
A biometria é solicitada em todas as vendas?
Não. O motor de risco utiliza IA para avaliar cada transação.
A biometria é acionada apenas quando padrões suspeitos são identificados.

[faq_whitelist_wallet]
É possível restringir saques para wallets autorizadas?
Sim. É possível habilitar whitelist de endereços, permitindo saques apenas para wallets previamente aprovadas.

Quais logs de segurança estão disponíveis?

[suporte_diferenca_ledger_fireblocks]
Qual a diferença entre Ledger e Fireblocks?
Ledger é o registro contábil interno (quem deve a quem).
Fireblocks é a custódia real dos ativos (onde o dinheiro está guardado e de onde ele sai).
Se houver saldo na Ledger mas não houver liquidez na Fireblocks, a transação pode falhar por falta de liquidez.

[suporte_erro_liquidez]
O que significa erro de liquidez?
Ocorre quando a operação é autorizada no sistema (Ledger), mas não há saldo disponível na Fireblocks no momento da execução.
A transação entra em falha por indisponibilidade de fundos reais na carteira operacional.

[suporte_valida_id_pending]
Por que um usuário pode cair em valida_id_pending mesmo sem atingir o mínimo de KYC?
O antifraude pode ser ativado se:
- A transação for maior que R$5.000,00;
- Houver duas transações do mesmo CPF em curto intervalo (aproximadamente 5 minutos).

[suporte_reabrir_kyc]
Se o usuário saiu da página durante o KYC, ele pode retornar?
Sim, se ele não tiver cancelado a operação, pode reabrir o link e continuar.
Caso tenha cancelado ou expirado, é necessário estornar e gerar novo QR Code.

[suporte_saque_fireblocks_procedimento]
O procedimento muda conforme o status do component.saque_cripto_fireblocks_to_merchant?
Sim.
Se estiver Pending, aguardar assinatura interna.
Se estiver Fail, adicionar wallet na whitelist, processar e remover da whitelist.
Se estiver Error, verificar saldo e reprocessar.

[templates_erro_template_nao_encontrado]
O que fazer quando a API retorna erro de template não encontrado?
1. Verificar se o nome do template enviado na rota POST /link está exatamente igual ao configurado no Dashboard.
2. Confirmar se não há espaços extras no início ou final do nome.
3. Validar se o template pertence ao ambiente correto (Sandbox ou Produção).
4. Se persistir, escalar para o time técnico.

[templates_erro_moeda_invalida]
O que fazer quando a API retorna erro de moeda ou rede inválida?
1. Confirmar se a moeda de contraparte está habilitada no Template.
2. Verificar se a rede selecionada é compatível com o ativo.
3. Validar se o ambiente suporta aquela moeda.
4. Caso esteja tudo correto, verificar logs da requisição.

[templates_quote_divergente]
O que fazer quando o valor exibido no checkout está diferente do esperado pelo merchant?
1. Verificar se está habilitada a opção "Cobrar Taxa do Cliente Final".
2. Verificar se está habilitada a opção "Cobrar Taxa de Rede do Cliente".
3. Confirmar se existe Split de Taxa (fixo ou percentual) configurado.
4. Recalcular o valor considerando taxas aplicadas.

[templates_erro_split]
O que fazer quando o Split de Taxas não está sendo creditado?
1. Confirmar se o Split está ativo no Template correto.
2. Verificar se o Template utilizado na API é o mesmo configurado.
3. Validar nos logs se o valor do split foi aplicado no quote.
4. Se não houver registro, escalar para análise técnica.

[templates_wallet_nao_exibida]
O que fazer quando a wallet de depósito não aparece no checkout?
1. Confirmar se a opção "Mostrar Wallet de Depósito" está ativada no Template.
2. Verificar se a operação é compatível com exibição de wallet (ex: offramp).
3. Testar com outro template para validar comportamento.

[templates_erro_unidades_minimas]
O que fazer quando o valor enviado na API está incorreto?
1. Confirmar se o valor foi enviado em unidade mínima.
   - Stablecoins devem ser enviadas em cents.
   - Bitcoin deve ser enviado em satoshis.
2. Verificar se houve conversão duplicada no backend do merchant.
3. Testar manualmente a requisição com valor fixo.

[templates_erro_cor_layout]
O que fazer quando o layout do checkout está quebrado?
1. Verificar se a cor do botão está em formato HEX válido.
2. Validar proporção do logo (entre 1:1 e 1:3).
3. Testar com logo padrão para isolar problema.

[templates_template_errado]
O que fazer quando o merchant afirma que o checkout está usando configuração errada?
1. Confirmar qual nome de template está sendo enviado na API.
2. Validar no Dashboard se aquele template está ativo.
3. Confirmar se não há cache no sistema do merchant.
4. Gerar novo link para teste.

[templates_procedimento_padrao_api]
Qual o procedimento padrão ao investigar erro relacionado a Template?
1. Coletar payload enviado pelo merchant.
2. Identificar nome do template utilizado.
3. Validar configurações no Dashboard.
4. Conferir logs da requisição.
5. Reproduzir erro em Sandbox.
6. Escalar apenas após confirmação técnica.

Documentos aceitos:

[kyc_tiers_pf]
Como funcionam os Tiers de KYC para Pessoa Física (CPF)?
Os limites são aplicados por CPF a cada 30 dias.

Tier 1:
Limite: R$ 10.000
Requisitos:
- Nome e CPF
- Validação automática via metadados do PIX ou formulário no checkout

Tier 2:
Limite: R$ 150.000
Requisitos:
- Documento
- Selfie com liveness

Tier 3:
Limite: Customizado
Requisitos:
- Declaração de capacidade financeira
- Análise manual em back-office

[kyc_fluxo_pf]
Como funciona o fluxo de KYC para CPF?
No primeiro uso, o cliente entra automaticamente no Tier 1.
O Tier 1 é validado automaticamente via PIX ou por formulário.
Ao atingir o limite do Tier 1, o cliente é solicitado a realizar KYC completo.
Após concluir o Tier 2, o limite passa para R$ 150.000.
Upgrades acima do Tier 2 são realizados manualmente pela equipe NoxPay.

[kyc_recomendacao_volumes_altos]
Qual a recomendação para usuários com volumes elevados?
Para volumes acima do Tier 2, recomenda-se realizar pré-registro com a equipe NoxPay.
Isso evita bloqueios durante o fluxo operacional.

[kyc_tiers_pj]

Tier 1:

[antifraude_funcionamento]
Como funciona o sistema antifraude da NoxPay?
Todas as transações passam pelo motor de risco da NoxPay.
Uma transação pode ser sinalizada independentemente do Tier do usuário.

[antifraude_flag]
O que acontece quando uma transação é sinalizada como fraude?
O CPF ou CNPJ é marcado como potencial fraude.
A marcação permanece ativa até nova validação.

[antifraude_remocao_flag]
Como remover uma flag de fraude?
Usuário Tier 1:
Deve realizar validação completa equivalente ao Tier 2.

Usuário Tier 2 ou superior:
Deve realizar nova selfie com liveness.

Empresas:
Devem validar o UBO.

[antifraude_observacao]
Os critérios de antifraude podem ser configurados pelo merchant?
Não.
Os critérios de classificação como fraude são definidos exclusivamente pelo motor de risco da NoxPay e não são configuráveis pelo merchant.

[faq_noxpay_fluxo_pagamento]
O que é a NoxPay e como funciona o fluxo de pagamento?
A NoxPay integra o sistema financeiro nacional (PIX) aos ativos digitais globais.
O fluxo ocorre em 4 etapas:
1. O usuário final lê o QR Code (com cotação fixada).
2. Se identificado risco, pode ser solicitada biometria facial durante o fluxo.
3. Após pagamento e validação, o saldo é creditado na Dashboard do lojista.
4. O lojista solicita o saque quando desejar para o endereço de carteira (wallet) cadastrado.

[faq_saque_tempo_blockchain]
Qual o tempo de liquidação do saque para a wallet?
O envio para a blockchain é realizado no ato da solicitação de saque na Dashboard.
O tempo médio para confirmação e chegada na carteira de destino é inferior a 5 minutos, dependendo da rede.

[faq_lentidao_blockchain]
O que ocorre em caso de lentidão na rede blockchain?
Utilizamos redes de alta performance que raramente congestionam.
Caso exista lentidão global na rede escolhida, o status pode ser acompanhado pelo Hash da transação disponível na Dashboard.

[faq_valor_divergente_pix]
O que acontece se o cliente pagar um valor diferente (maior ou menor) do que o estipulado no QR Code PIX?
A NoxPay utiliza QR Codes Dinâmicos onde o valor é imutável.
O aplicativo bancário do usuário não permite editar o valor a ser pago, impedindo pagamento maior ou menor na origem.

Pessoa Física (CPF) pode abrir conta na NoxPay?
Não.
Aceitamos apenas empresas dentro dos critérios de elegibilidade definidos.

[faq_taxas_cobradas]
Quais são as taxas cobradas pela NoxPay?
Trabalhamos com um modelo negociável, considerando principalmente:
- O faturamento mensal da empresa
- O perfil de risco do cliente

Condições de referência:
- Até R$ 100 mil/mês: taxa padrão de 2,5%
- Acima de R$ 100 mil/mês: taxa negociável conforme volume e risco

[faq_como_sacar_valor_conta]
Como posso sacar o valor da minha conta?
Os valores disponíveis podem ser movimentados conforme a estratégia do lojista:
- Envio para carteira própria (wallet)
  - Cold wallet: armazenamento offline (maior segurança)
  - Hot wallet: carteira online (mais praticidade)
- Envio para exchanges (ex.: Binance, Bybit, Coinbase)
- Outras formas aplicáveis (ex.: cartão cripto, quando disponível)

Antes de sacar, é importante garantir saldo suficiente para cobrir as taxas de rede.

[docs_crossramp_introducao]
O que é o Crossramp e qual a proposta do produto?
O Crossramp permite conversão direta entre FIAT (ex.: BRL) e cripto (ex.: USDT) usando PIX como método de pagamento.
Via API, é possível gerar links de checkout personalizados, processar KYC quando necessário, acompanhar status em tempo real e receber notificações via webhooks.
A jornada é completa — da cotação à confirmação — com foco em segurança, transparência e usabilidade.

[docs_checkout_componente_unico]
Por que o checkout não pode ser fragmentado (somente PIX, somente cotação, etc.)?
O checkout foi projetado como uma experiência única para garantir conformidade regulatória e manter o fluxo do usuário íntegro.
Fragmentar partes (ex.: só PIX ou só cotação) pode quebrar o fluxo e gerar experiências interrompidas, especialmente quando há KYC, pagamento e liquidação no meio do processo.

[docs_endpoint_link_retorna_url]
O que o endpoint POST /link retorna?
A resposta do POST /link retorna a URL do checkout gerado.
Essa URL é usada para redirecionamento ou para incorporar (embed) o checkout na aplicação.

[docs_fluxo_integracao_visao_geral]
Qual é o fluxo de integração (visão geral) do Crossramp?
1. Receber API Key e acesso ao Dashboard (enviado no e-mail do onboarding).
2. Configurar um Template.
3. Definir os processos que fazem sentido pro negócio: onramp, onramp-instant, offramp e/ou offramp-instant.
4. Gerar o checkout chamando POST /link.
5. Acompanhar o processo via webhooks, Dashboard e/ou métodos GET.
Com isso, a integração fica pronta: um endpoint, múltiplos fluxos.

[docs_validade_checkout_10min]
Qual é a validade do checkout e em que momento ela deixa de se aplicar?
O checkout tem validade de 10 minutos entre a cotação e o depósito (via PIX ou cripto).
Após o depósito ocorrer, essa expiração deixa de ser aplicada.

[docs_checkout_redirecionamento]
Como funciona a implementação por redirecionamento para o checkout?
1. Definir os valores (em FIAT ou Cripto) que iniciarão o checkout.
2. Informar a URL de retorno da sua aplicação (para redirecionamento ao final).
3. Criar o checkout via API e receber a URL do checkout.
4. Redirecionar o usuário para essa URL (ou usar em um botão/CTA).
5. Ao final, o usuário volta para a URL de retorno.

[docs_checkout_embed_iframe]
Como funciona a implementação incorporando (embed) o checkout?
1. Definir os valores (em FIAT ou Cripto) que iniciarão o checkout.
2. Criar o checkout via API e receber a URL do checkout.
3. Incorporar a URL na sua aplicação (ex.: embed/iFrame), mantendo o checkout “inteiro” dentro do seu app.

[docs_embed_boas_praticas]
Quais boas práticas usar ao incorporar (embed) o checkout?
- Testar layout em diferentes páginas, resoluções e tamanhos de janela.
- Testar em múltiplos navegadores para garantir consistência.
- Evitar tentar “extrair” partes do checkout (somente uma etapa/tela), porque isso quebra o fluxo e causa problemas de experiência.

[processo_onramp_definicao_detalhada]
O que é o processo Onramp (detalhado) e como ele liquida?
Onramp é entrada via PIX (BRL) e saída em cripto, sem liquidação imediata.
Ao usar o parâmetro "onramp" no POST /link, ao final do processo o valor convertido (e eventuais taxas cobradas do cliente) fica disponível como saldo na NoxPay.
Esse saldo pode ser sacado depois via Dashboard, conforme conveniência.

[processo_onramp_telas_criticas]
Quais telas do Onramp são obrigatórias para manter o fluxo funcionando?
É crucial manter o checkout disponível nas etapas em que o usuário pode precisar agir.
Telas críticas:
- Cotação (Quoting)
- Depósito via PIX
- Validação de identidade (ID validation), quando aplicável
Se essas telas não forem exibidas, o fluxo pode ser interrompido.

[processo_onramp_fluxo_financeiro_macro]
Como funciona o fluxo financeiro do Onramp ao longo das etapas?
- Até o depósito via PIX: não há movimentação para o merchant.
- Após o pagamento: o BRL é creditado no saldo do cliente na Nox (ainda não significa valor disponível pro merchant).
- Após KYC/antifraude (se necessário): os valores seguem no saldo do cliente até a conversão.
- Na conversão: BRL do cliente vira cripto e é transferido para a conta do merchant na Nox; taxas do merchant (se configuradas) também são creditadas; taxas da NoxPay são descontadas nesse momento.
- Em caso de cancelamento: inicia fluxo de reembolso; no fim, o BRL é devolvido via PIX e nenhuma taxa é coletada (nem merchant nem Nox) quando termina em reembolso.

[processo_offramp_definicao_detalhada]
O que é o processo Offramp (detalhado) e qual a condição de falha?
Offramp é saída: entrada de cripto e saída em BRL via PIX, liquidada a partir do saldo em cripto do merchant.
Ao usar "offramp" no POST /link, o checkout coleta dados e processa o PIX de saída financiado pelo saldo do merchant.
O processo falha se o saldo em cripto do merchant não for suficiente para cobrir o valor da transação.

[processo_offramp_regra_cpf_pix]
Qual regra crítica existe no Offramp sobre CPF/CNPJ e titularidade da chave PIX?
O PIX de saída só deve ser processado se o CPF/CNPJ informado corresponder ao titular da conta bancária vinculada à chave PIX.

[processo_offramp_telas_criticas]
Quais telas do Offramp são obrigatórias para manter o fluxo funcionando?
É crucial manter o checkout disponível nas telas em que o usuário precisa agir.
Telas críticas:
- Cotação (Quoting)
- Autoidentificação KYC e formulário de chave PIX
- Validação de identidade (ID validation), quando aplicável
- Tela de depósito de cripto
A não exibição dessas telas pode interromper o fluxo.

[processo_onramp_instant_definicao]
O que é o Onramp Instant e qual a recomendação de rede?
Onramp Instant é entrada via PIX (BRL) e saída em cripto com liquidação imediata.
Ao usar "onramp-instant" no POST /link, a cripto é enviada imediatamente para a carteira informada ao final do processo.
Esse fluxo envolve taxas de rede (Network Fees). Se o template não repassar ao cliente, elas são descontadas do saldo do merchant.
Recomendação: preferir redes mais baratas para evitar custo excessivo.

[processo_onramp_instant_saque_cripto]
Como funciona o “Processamento do Saque de Criptomoeda” no Onramp Instant?
Na etapa de saque, a transação de cripto é enviada para a carteira definida na criação do checkout.
O valor é bloqueado no saldo para realizar o saque, incluindo taxas de rede.
Se falhar (rede/endereço incompatível ou outro motivo), o saldo é desbloqueado e a transação fica suspensa.
O merchant pode solicitar saque manual pelo Dashboard ou acionar o time para ajustar e reprocessar.

[processo_offramp_instant_definicao]
O que é o Offramp Instant e como funciona o depósito via checkout?
Offramp Instant é saída: entrada de cripto e saída em BRL via PIX, onde o depósito de cripto ocorre no próprio checkout.
Ao usar "offramp-instant" no POST /link, o checkout solicita depósito de cripto para que o PIX seja processado.
Esse fluxo pode ser configurado no Template para ocultar o endereço de depósito do cliente; nesse caso, endereço e valor são enviados via webhook.

[processo_offramp_instant_valor_exato_deposito]
Qual é a regra do valor exato de depósito no Offramp Instant?
O valor depositado deve corresponder exatamente ao EXACT_deposit_amount enviado no webhook (ou ao valor exibido na tela).
Valores divergentes não são reconhecidos automaticamente.
Nessa situação, é necessário acionar o time da NoxPay para prosseguir.

[faq_prazo_reembolso]
Qual o prazo para reembolso?
O prazo para reembolso depende da natureza da ocorrência e da etapa em que a transação se encontra.
Em casos de fraude comprovada, golpe ou falha operacional, a devolução poderá ser realizada após análise do time de Compliance, respeitando os prazos do Mecanismo Especial de Devolução (MED), quando aplicável.
Nos casos de desacordo comercial entre as partes, o reembolso deve ser tratado diretamente entre elas, pois a NoxPay não realiza estornos após a liquidação da operação.

[faq_publico_alvo_noxpay]
Para quem é o NoxPay?
O NoxPay Crossramp é indicado para empresas que precisam integrar o PIX brasileiro a fluxos de criptoativos ou moedas múltiplas, como:
- Exchanges e corretoras de criptomoedas
- PSPs e gateways de pagamento
- Plataformas SaaS e marketplaces
- Empresas globais que vendem para ou realizam pagamentos a usuários no Brasil

[faq_quando_usar_cada_fluxo]
Qual fluxo devo usar em cada situação?
Onramp:
Adicionar saldo em criptomoeda para uma plataforma ou usuário utilizando PIX.

Onramp Instant:
Entregar criptomoeda diretamente na carteira externa do usuário após o pagamento via PIX.

Offramp:
Realizar pagamentos em BRL a partir de criptomoedas já disponíveis no saldo Nox (ex.: folha de pagamento, saques).

Offramp Instant:
Permitir que usuários vendam suas criptomoedas e recebam BRL via PIX, utilizando um endereço único de depósito.

[faq_jornada_checkout_padrao]
Como funciona a jornada padrão do checkout?
Para fluxos onramp e onramp instant, o usuário percorre:
- Início: clique em “Pagar com PIX / NoxPay”
- Identificação: preenchimento de nome completo e CPF (deve corresponder ao titular da chave PIX)
- Cotação: exibição de valores, taxa de câmbio e taxas aplicáveis
- Pagamento via PIX: QR Code e opção copia e cola
- Tela de sucesso: resumo da operação e return_url
- Webhook: envio do status final ao merchant

Nos fluxos de offramp, podem existir etapas adicionais como formulário de chave PIX ou depósito de criptomoeda.

[faq_diferenca_fluxos_offramp]
O que muda nos fluxos Offramp e Offramp Instant?
Offramp:
- Usuário informa nome, CPF e chave PIX
- Confirma valor em BRL
- Aguarda pagamento via PIX realizado pela Nox

Offramp Instant:
- Visualiza endereço de depósito
- Visualiza rede suportada
- Visualiza valor EXATO de criptomoeda a ser enviado
Após confirmação on-chain do valor correto, a Nox executa o pagamento via PIX e exibe a tela de sucesso.

[faq_status_substatus_definicao]
O que são status e substatus?
Cada transação possui dois níveis:
Status: etapa principal do fluxo (ex.: quoting, pix_deposit, crypto_withdrawal, client_side_success).
Substatus: detalhamento da etapa (ex.: INITIAL, QRCODE, DONE, ERROR, EXPIRED).

Essas informações estão disponíveis no endpoint GET /checkout/{end2end} e nos webhooks.

[faq_significado_etapas_status]
O que cada etapa principal de status representa?
quoting — Visualização/aceite da cotação
kyc_validation — Coleta de CPF/PIX
id_validation — Validação de documento e selfie
pix_deposit — Aguardando pagamento via PIX
pix_withdrawal — Processamento de saída em BRL via PIX
crypto_deposit — Aguardando depósito de criptomoeda
crypto_withdrawal — Envio de criptomoeda
swap_fiat_for_crypto / swap_crypto_for_fiat — Conversão entre FIAT e cripto
client_side_success — Tela final de conclusão

[faq_quando_pagamento_finalizado]
Quando devo considerar um pagamento como finalizado?
Um pagamento pode ser tratado como finalizado quando:
- status = client_side_success e substatus = INITIAL, ou
- A última etapa crítica (pix_withdrawal ou crypto_withdrawal) estiver com substatus = DONE.

Merchants normalmente mapeiam esses estados para:
PENDING
COMPLETED
FAILED

Por quanto tempo uma cotação permanece válida?

O substatus DIVERGENT ocorre quando o titular da chave PIX não corresponde ao CPF/CNPJ registrado.
Nesse caso, o pagamento é interrompido.

[faq_iof]
Há incidência de IOF nas operações da NoxPay?
Não.
Os fluxos de conversão de criptomoedas processados dentro da NoxPay não estão sujeitos à incidência de IOF.

[faq_imposto_ganho_capital]
Há incidência de impostos sobre cripto durante a transação?
Não.
A tributação ocorre apenas quando há ganho de capital.
Se o cliente compra a cripto pelo preço Y e utiliza pelo mesmo preço Y, não há ganho de capital nem fato gerador de imposto.

[faq_modelo_regulatorio_fluxo]
Qual é a estrutura regulatória do fluxo de pagamento?
O modelo segue:
1. Cliente paga a NoxPay via PIX
2. A Nox credita BRL na conta do cliente
3. A Nox converte o saldo em criptomoeda
4. A criptomoeda é transferida da conta do cliente para o merchant

Esse modelo garante atribuição correta conforme a IN 1.888.

[faq_relatorio_in_1888_merchant]
Os merchants precisam reportar transações com cripto conforme a IN 1.888?
Não.
A NoxPay é responsável por realizar todos os reportes obrigatórios à Receita Federal.

[faq_documento_fiscal_cliente]
O cliente recebe algum documento fiscal diretamente?
Não.
As declarações são transmitidas diretamente à Receita Federal, sem envio de documento fiscal adicional ao cliente.

[faq_recebimento_cripto_evento_tributavel]
O recebimento de criptomoeda é um evento tributável para o merchant?
Não.
Apenas haverá tributação caso exista venda posterior da criptomoeda com ganho de capital.
O simples recebimento não gera fato tributável.
